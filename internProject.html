Privacypass by Simulpass

UI design for an app made to change how we share data.

OVERVIEW

In the summer of 2020 I interned at Simulpass, a startup providing a blockchain-based platform for making professional and personal connections. My project was to work with a team of other UX interns to optimize the app's interface for iOS and redefine the app's visual identity through its iconography and brand.

MY CONTRIBUTIONS

I worked on a team of 3 undergraduate UX/UI interns. Primarily, I optimized the UX and UI design of the "Maps" tab of the app for iOS compatibility. I also was responsible for developing my designs using SwiftUI. Additionally, I worked with another intern to reimagine the app's brand design.

OBJECTIVE

What is Simulpass?

Simulpass is a startup with the intention of creating a more secure way to store and share data online. Its blockchain-based platform and distributed network is made for making professional and personal connections, and being able to safely share your personal information with those connections.

MAJOR PROJECTS

How can we improve?

My area of focus within the interface redesign was on two task flows of the app's Maps feature. Maps is meant for the user to be able to "check in" to their current location and see a map with connections that are also "checked in" near them. Maps contains a travel log for the user to record past, current, and future locations, a check in interface, and a real-time map display. My focus was on the check in and travel log task flows. In addition to UX work, I also contributed to building Simulpass's visual and brand identity.

How do we create a visual identity?

My team established a style guide to use throughout the redesign of the app and to ground the app icon redesign on. Title screen, typography, color scheme and iconography included.

For the icon redesign I created three iterations of new app icons, each designed to play on the original icon's "S" silhouette and stay true to the connecting mission of Simulpass by using two interlocking pieces. A gradient was used in place of solid color to create a deeper visual complexity in the absence of the text that previously followed the icon.

Where can we most improve the user's experience?

WIREFRAMING

The first step towards improving the user experience was to identify any task flow pain points and visual discontinuities. I approached the process by running a personal "think aloud" study to gauge my own expectations and understanding of the interface.

The insights gathered were then taken into account during the wireframing process. My observations can be summarized into three main points. From here, I reimagined both the interface and task flows.

1 Counter intuitive navigation.

Accessing the check in and travel log screens via buttons, implies that the three Maps task flows are not accessible from each other and that user will not need to flip from one to the other: this is not the case.

2 Visually convoluted.

Placing all three taskflow starting points in a vertical stack is an inefficient use of screen space. It takes space from what would otherwise be the user map and makes it easy for the user to mis-click and accidentally enter a different task flow.

3 Misplaced emphasis.

Placing the non-map task flows in bolded buttons visually misconstrues their significance to the current task flow and draws the user away from the map.

SWIFTUI

After creating high-fidelity wireframes of my designs I was asked to implement the screens using SwiftUI. The key differences between the former UX design and my redesign are summarized below.

Developing my designs allowed me to better understand the limitations and best practices of iOS design. Also significant was the process of learning a new coding language.

  • Simple task flows.

    • The previous button interface was swapped for a tab bar that displays all options and highlights the current position. All task flows are now accessible from all others at any point in any flow.

  • iOS compatible interface.

    • The UI and main functions of each flow: date picker, list with slide to delete capability, and confirmation messages are compatible with iOS devices.

  • Cohesive visual identity.

    • UI elements reflect the app's style guide at each point in the task flows. Elements of each flow are visually emphasized or deemphasized accordingly.
